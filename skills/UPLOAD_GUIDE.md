# AgentScope PaaS 技能上传指南

## 概述

本系统支持用户通过前端页面上传技能配置，提供灵活的技能管理方式。所有技能配置遵循 **skill-creator** 规范，确保技能的标准化和可维护性。

## 支持的上传方式

### 1. 单个 SKILL.md 文件上传
- **文件格式**：`.md` 文件
- **大小限制**：最大 3MB
- **文件要求**：必须符合 skill-creator 规范
  - YAML frontmatter（包含 name 和 description）
  - Markdown body（技能指令）

### 2. 文件夹上传
- **大小限制**：最大 3MB
- **文件数量限制**：最多 50 个文件
- **必需文件**：`SKILL.md`
- **可选目录**：
  - `scripts/` - 可执行脚本
  - `references/` - 参考文档
  - `assets/` - 资源文件

### 3. ZIP 压缩包上传
- **文件格式**：`.zip`
- **大小限制**：最大 3MB
- **文件数量限制**：最多 50 个文件
- **必需文件**：`SKILL.md`（在根目录）
- **可选目录**：同文件夹上传

## SKILL.md 文件规范

### 必需结构

```markdown
---
name: skill-name
description: 技能描述，说明何时触发及功能
---

# 技能标题

## 技能内容
（技能的具体指令和说明）
```

### YAML Frontmatter 字段

#### 必需字段
- **name** (string): 技能标识符
  - 格式：小写字母、数字、连字符
  - 最大长度：100字符
  - 示例：`order-processing`, `text-analysis`

- **description** (string): 技能描述
  - 最小长度：50字符
  - 最大长度：500字符
  - 内容：说明何时触发、技能功能
  - 示例：`订单处理技能，提供订单创建、查询、修改、取消功能。当用户需要处理电商订单时使用此技能。`

#### 可选字段
- **compatibility** (string): 兼容性说明
  - 说明技能的依赖和权限要求

### Markdown Body 要求

- **最小长度**：100字符
- **最大长度**：20,000字符
- **推荐章节**：
  - ## 概述
  - ## 功能特性
  - ## 使用方法
  - ## 配置要求
  - ## 示例

## 上传文件要求

### 文件大小限制
- 单个文件：最大 3MB
- 总上传大小：最大 10MB
- ZIP压缩包：最大 3MB

### 文件类型限制

**允许的文件类型**：
- `.md` - Markdown文档
- `.py` - Python脚本
- `.js` - JavaScript脚本
- `.txt` - 文本文件
- `.json` - JSON配置文件
- `.zip` - ZIP压缩包

**禁止的文件**：
- `.env` - 环境变量文件
- `credentials.json` - 凭证文件
- `secret.txt` - 密钥文件

### 安全检查

系统会自动进行以下安全检查：
1. **恶意代码检测**：扫描可疑代码模式
2. **路径遍历防护**：防止恶意路径访问
3. **文件名验证**：确保文件名符合规范
4. **内容安全**：检查禁止的内容模式

```
    └── config.json       # 配置文件
```

## 上传流程

### 前端上传界面

1. **选择上传方式**
   - 单个文件上传
   - 文件夹上传
   - ZIP压缩包上传

2. **文件选择**
   - 根据选择的方式，选择相应的文件或文件夹

3. **自动验证**
   - 文件大小检查
   - 文件格式验证
   - SKILL.md 规范验证
   - 安全扫描

4. **预览确认**
   - 显示技能信息
   - 显示文件列表
   - 显示验证结果

5. **上传处理**
   - 解析 SKILL.md
   - 提取元数据
   - 保存到系统

### 验证规则

#### YAML Frontmatter 验证
```yaml
# ✅ 正确示例
---
name: order-processing
description: 订单处理技能，提供订单管理功能
---

# ❌ 错误示例
---
name: Order Processing  # 包含空格和大写字母
description: 太短      # 少于50字符
---
```

#### 内容质量验证
- **清晰度检查**：确保指令清晰易懂
- **完整性检查**：包含必要的使用说明
- **格式检查**：Markdown格式正确

## 错误处理

### 常见上传错误

#### 1. 文件过大
```
错误：文件大小超过3MB限制
解决：压缩文件或减少文件内容
```

#### 2. 格式错误
```
错误：SKILL.md 缺少必需的 YAML frontmatter
解决：确保文件以 --- 开始的 YAML 块
```

#### 3. 字段验证失败
```
错误：name 字段格式不正确
解决：使用小写字母、数字和连字符
```

#### 4. 安全检查失败
```
错误：检测到可疑代码模式
解决：移除可疑代码或联系管理员
```

## 技能管理

### 上传后的操作
1. **技能预览**：查看技能详细信息
2. **技能测试**：测试技能功能
3. **技能编辑**：修改技能配置
4. **技能删除**：删除不需要的技能

### 技能版本控制
- 系统自动为每个上传的技能创建版本
- 支持技能回滚到之前版本
- 保留技能修改历史

## 最佳实践

### 1. 技能设计原则
- **单一职责**：每个技能专注一个功能
- **清晰描述**：description 字段要详细准确
- **完整指令**：提供充分的使用说明
- **示例丰富**：包含多个使用示例

### 2. 文件组织
- **结构清晰**：合理组织 scripts、references、assets
- **命名规范**：使用有意义的文件名
- **文档完善**：提供详细的参考文档

### 3. 安全考虑
- **敏感信息**：不要在技能中包含密钥或密码
- **权限最小化**：只申请必要的权限
- **输入验证**：验证所有用户输入
