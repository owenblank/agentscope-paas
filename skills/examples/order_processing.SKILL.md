# 订单处理技能

## 概述

此技能提供完整的订单管理功能，包括订单的创建、查询、修改和取消操作。适用于电商平台、订单管理系统等场景。

## 功能特性

### 订单创建
根据用户信息和商品列表创建新订单，支持：
- 多种支付方式（支付宝、微信支付、信用卡、银行转账）
- 灵活的收货地址配置
- 自动计算订单总金额
- 生成唯一订单编号

### 订单查询
通过订单ID或订单编号查询订单详情，包括：
- 订单基本信息
- 商品详细信息
- 物流跟踪信息
- 支付状态

### 订单修改
修改已创建订单的信息：
- 修改收货地址
- 调整商品数量
- 更新订单备注

### 订单取消
取消指定订单并处理退款：
- 多种取消原因支持
- 自动退款流程
- 退款状态跟踪

## 使用方法

### 创建订单
```
输入：
- 客户ID
- 商品列表（商品ID、数量、单价）
- 收货地址信息
- 支付方式

输出：
- 订单ID和订单编号
- 订单状态
- 订单总金额
- 预计送达时间
```

### 查询订单
```
输入：
- 订单标识（ID或编号）
- 是否包含详细信息

输出：
- 订单基本信息
- 商品详情
- 物流信息
- 支付信息
```

### 修改订单
```
输入：
- 订单ID
- 修改类型（地址/商品/备注）
- 新的值
- 修改原因

输出：
- 修改成功状态
- 更新后的订单信息
```

### 取消订单
```
输入：
- 订单ID
- 取消原因
- 退款方式

输出：
- 取消成功状态
- 退款信息
- 预计退款时间
```

## 配置要求

### API配置

- 认证：需要Bearer Token或API Key
- 超时：建议30秒

### 权限要求
- 订单创建权限
- 订单查询权限
- 订单修改权限
- 订单取消权限

## 参数说明

### 创建订单参数
- `customer_id` (string, 必需): 客户唯一标识符
- `products` (array, 必需): 商品列表
  - `product_id` (string): 商品ID
  - `quantity` (integer): 购买数量，最小值为1
  - `price` (number): 商品单价，非负数
- `shipping_address` (object, 必需): 收货地址
  - `recipient_name` (string): 收件人姓名
  - `phone` (string): 联系电话
  - `province` (string): 省份
  - `city` (string): 城市
  - `district` (string): 区县
  - `detailed_address` (string): 详细地址
  - `postal_code` (string): 邮政编码（可选）
- `payment_method` (string, 必需): 支付方式
  - 可选值：alipay, wechat_pay, credit_card, bank_transfer
- `remarks` (string, 可选): 订单备注，最大500字符

### 查询订单参数
- `order_identifier` (string, 必需): 订单ID或订单编号
- `identifier_type` (string, 可选): 标识类型
  - 可选值：order_id, order_number
  - 默认值：order_id
- `include_details` (boolean, 可选): 是否包含详细信息
  - 默认值：true

## 错误处理

### 常见错误码
- `INVALID_PARAMS`: 参数验证失败
- `ORDER_NOT_FOUND`: 订单不存在
- `INSUFFICIENT_STOCK`: 库存不足
- `PAYMENT_FAILED`: 支付失败
- `ORDER_CANCELLED`: 订单已取消
- `PERMISSION_DENIED`: 权限不足

### 重试策略
- 网络错误：最多重试3次，采用指数退避
- 参数错误：不重试，需修正参数
- 权限错误：不重试，需检查权限配置

## 性能考虑

- **响应时间**：订单查询 < 500ms，订单创建 < 2s
- **并发限制**：每个用户最多10个并发订单操作
- **缓存策略**：订单信息缓存30分钟

## 安全注意事项

1. **数据验证**：严格验证所有输入参数
2. **权限检查**：确保用户有相应操作权限
3. **敏感信息**：不要在日志中记录完整支付信息
4. **防重复提交**：使用幂等性令牌防止重复创建订单

## 故障排查

### 订单创建失败
1. 检查商品库存是否充足
2. 验证收货地址格式是否正确
3. 确认支付方式是否可用
4. 检查客户账户状态

### 订单查询失败
1. 确认订单ID或编号是否正确
2. 检查是否有查询权限
3. 验证订单是否属于当前客户
