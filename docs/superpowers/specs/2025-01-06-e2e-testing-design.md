# AgentScope PaaS 端到端测试设计方案

**日期**: 2025-01-06
**项目**: AgentScope PaaS 平台
**设计者**: Claude Code
**状态**: 已批准

## 设计目标

设计并实施完整的端到端测试方案，验证 AgentScope PaaS 平台的核心业务流程，确保系统功能正常、性能达标、用户体验良好。

## 测试范围

### 核心业务流程
1. **用户认证流程**
   - 用户注册
   - 用户登录
   - 会话管理

2. **智能体管理流程**
   - 智能体创建
   - 配置表单填写
   - 智能体启动

3. **聊天交互流程**
   - 聊天界面交互
   - 消息发送接收
   - 错误处理

### 技术覆盖范围
- 前端 UI 组件
- 后端 API 接口
- 数据持久化
- 错误处理机制
- 性能指标

## 实施方案

### 阶段一：环境准备与检查

**目标**: 确保测试环境就绪

**步骤**:
1. 依赖检查
   - Python 环境 (3.8+)
   - Playwright 及浏览器驱动
   - pytest 测试框架
   - 其他必需依赖包

2. 服务启动
   - 后端 API 服务器 (http://localhost:8000)
   - 前端开发服务器 (http://localhost:3005)
   - 服务健康检查

3. 测试用户配置
   - 默认测试用户: `test@example.com` / `Test123456`
   - 用户不存在时的自动创建机制

**验证标准**:
- 所有依赖包正确安装
- 服务可正常访问
- 测试用户凭证有效

### 阶段二：测试配置与优化

**目标**: 配置测试参数，优化测试体验

**配置项**:
1. 测试环境配置
   ```json
   {
     "test_environment": {
       "frontend_url": "http://localhost:3005",
       "backend_url": "http://localhost:8000",
       "timeout": 5000
     }
   }
   ```

2. 测试用户配置
   ```json
   {
     "test_user": {
       "email": "test@example.com",
       "password": "Test123456"
     }
   }
   ```

3. 测试行为配置
   ```json
   {
     "test_settings": {
       "headless": false,
       "browser": "chromium",
       "screenshot_directory": "e2e_screenshots",
       "report_directory": "e2e_reports"
     }
   }
   ```

**优化措施**:
- 显示浏览器窗口便于观察测试过程
- 自动截图保存关键步骤
- 详细日志记录
- 超时和重试机制

### 阶段三：测试执行与监控

**目标**: 执行完整测试，实时监控结果

**执行命令**:
```bash
python e2e/main_test_runner.py e2e --interactive
```

**测试覆盖**:
1. **认证模块**
   - 用户注册流程
   - 用户登录流程
   - 登录状态保持
   - 退出登录

2. **智能体模块**
   - 智能体创建流程
   - 配置表单验证
   - 智能体列表显示
   - 智能体详情查看

3. **聊天模块**
   - 聊天界面加载
   - 消息发送功能
   - 消息接收显示
   - 错误处理

**监控指标**:
- 测试通过率
- 单个测试执行时间
- 页面响应时间
- 错误类型分布
- UI 渲染正确性

### 阶段四：结果分析与报告

**目标**: 分析测试结果，生成详细报告

**分析内容**:
1. 测试结果汇总
   - 总测试用例数
   - 通过/失败数量
   - 通过率百分比

2. 失败分析
   - 失败测试详情
   - 失败原因分类
   - 相关截图和日志

3. 性能评估
   - 平均响应时间
   - 页面加载时间
   - API 调用性能

4. 问题分类
   - **功能性问题**: 业务逻辑错误
   - **UI/UX 问题**: 界面显示或交互问题
   - **性能问题**: 响应慢或资源占用高
   - **配置问题**: 环境或参数配置错误

**报告格式**:
1. 控制台输出
   - 实时进度显示
   - 关键指标摘要
   - 通过/失败统计

2. JSON 报告文件
   - 详细测试结果
   - 时间戳记录
   - 错误堆栈信息

3. 截图集合
   - 关键步骤截图
   - 错误现场截图
   - 按时间顺序组织

4. 改进建议
   - 短期修复建议（高优先级）
   - 长期优化方向（中低优先级）
   - 测试覆盖率提升建议

## 技术架构

### 测试框架
- **主测试运行器**: `e2e/main_test_runner.py`
- **浏览器自动化**: Playwright (Chromium)
- **测试框架**: pytest
- **断言库**: pytest assertions

### 文件结构
```
e2e/
├── main_test_runner.py           # 主测试运行器
├── complete_agent_chat_e2e.py   # 完整E2E测试脚本
├── frontend_ui_test.py          # 前端UI测试
├── test_config.json             # 测试配置文件
└── e2e_screenshots/            # 测试截图目录
```

### 数据流
```
用户触发 → 主测试运行器 → 环境检查 → 服务启动 → 测试配置 →
浏览器测试 → 结果收集 → 报告生成 → 用户反馈
```

## 风险与缓解

### 潜在风险
1. **服务启动失败**
   - 缓解: 提供详细错误信息和启动脚本

2. **测试用户失效**
   - 缓解: 自动创建或注册流程

3. **浏览器驱动问题**
   - 缓解: 自动安装和版本检查

4. **网络超时**
   - 缓解: 可配置超时和重试机制

### 应急方案
1. 服务无法启动时，提供手动启动指南
2. 测试失败时，保存详细日志便于调试
3. 浅层测试作为快速验证备选方案
